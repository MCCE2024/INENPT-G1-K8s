Die Power Rangers & ihre Mission

Die Story

Ein Startup mit einer genialen SaaS-Idee ruft die Power Rangers zu Hilfe: Multi-Tenancy ist (noch) nicht möglich, aber die Infrastruktur muss sofort bereitstehen! Die Zeit drängt, die Konkurrenz schläft nicht – und nur die Rangers können mit Terraform, Kubernetes und ArgoCD eine sichere, skalierbare Plattform erschaffen.
“It’s Morphin Time – für DevOps!”

🦸‍♂️ Patrick (Green Ranger) - Der Retter
🧑‍🚀 Thomas (Zordon) - Lektor & Mentor

Die Mission

  • Mindestens 3 Services, einer mit Datenbank
  • Authentifizierung via OAuth2
  • Multi-Tenancy durch Infrastruktur, nicht Software
  • Jeder neue Tenant: 1 Git Commit genügt!
  • Mindestens ein Service läuft auf einer skalierbaren VM
  • Cloud: Exoscale oder AWS
  • IaC: Terraform, GitOps: ArgoCD
  • Security: Von Anfang an mitgedacht
BereichTechnologien
FrontendConsumer Web App (Node.js)
BackendAPI Service (Node.js)
Data ProducerPython CronJob
InfrastructureArgoCD, Helm, Terraform

Zord Battle – Constraints & Shortcomings

⚔️ Die Regeln des Kampfes (Projekt-Constraints)

  • 🖱️ No Clicks: Alles muss automatisiert, keine manuelle Einrichtung!
  • ☸️ Kubernetes Only: Alles läuft im Cluster (außer ein VM-Service)
  • 🌩️ Cloud: Exoscale oder AWS
  • 🔄 Tenant Onboarding: Ein Git Commit reicht
  • 🏗️ IaC & GitOps: Terraform & ArgoCD sind Pflicht
  • 🛡️ Security: Von Anfang an mitgedacht
  • ⚡ Scalability: Mindestens ein Service auf skalierbarer VM

Wie die Power Rangers gegen Monster kämpfen, so kämpfen wir gegen die Herausforderungen der Cloud!
